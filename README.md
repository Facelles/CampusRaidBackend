# 🚀 CampusRaid Backend (Bun + Express + Prisma)

CampusRaid — це гейміфікована освітня платформа для студентів та викладачів, де вивчення програмування перетворюється на битви з IT-монстрами (босами). 

Архітектура побудована за принципом **«швидко в розробці, легко деплоїться, безпечно типізується»**. Замість класичного стеку Node.js ми використовуємо **Bun**. Він неймовірно швидкий і має вбудований TypeScript.

---

## 🛠️ Технологічний Стек

- **Runtime**: [Bun](https://bun.sh/)
- **Фреймворк**: Express.js
- **База Даних**: PostgreSQL (NeonDB / Supabase)
- **ORM**: Prisma
- **Валідація**: Zod

---

## 🏗️ Швидкий Старт

1. **Встановлення залежностей:**
   ```bash
   bun install
   ```
2. **Налаштування середовища:**
   Створіть файл `.env` на основі `.env.example` та вкажіть ваш `DATABASE_URL`.
3. **Синхронізація бази даних та наповнення:**
   ```bash
   bun run db:push
   bun run prisma/seed.ts
   ```
   *(Команда `seed.ts` створить базовий університет, товари для магазину та 3-х унікальних босів з 4 рівнями завдань кожен).*
4. **Запуск сервера (в режимі розробки):**
   ```bash
   bun run dev
   ```

## 🛣️ Специфікація API Ендпоінтів

Усі запити виконуються до `/api/...`. Всі тіла запитів (body) повинні бути у форматі `application/json`.

### 🔑 Блок 1: Авторизація
- **`POST /api/auth/login`**
  - *Опис:* Вхід або реєстрація користувача. Якщо користувач новий, він автоматично створюється.
  - *Body:* `{ "email": "student@knu.ua", "name": "Влад", "universityId": "..." }`
  - *Відповідь:* Об'єкт користувача.

### 👾 Блок 2: Ігрова логіка (Рейд на Боса)
- **`GET /api/boss/active`**
  - *Опис:* Повертає поточного активного Боса, поточний квест-пазл (`Puzzle`) та перемішані блоки коду.
- **`GET /api/boss/join?inviteCode=...`**
  - *Опис:* Дозволяє приєднатися до приватного боса, створеного викладачем, використовуючи унікальний код доступу (наприклад, `HACK-ЗЕЛ`).
- **`POST /api/boss/attack`**
  - *Опис:* Спроба студента розв'язати код. Зменшує HP боса при успіху, нараховує XP та монети. Також зберігає спробу в `BossAttempt` для статистики викладача.
  - *Body:* `{ "userId": "...", "puzzleId": "...", "blockIds": ["b1", "b3", "b2", "b4"] }`
  - *Відповідь:* `{ "success": true/false, "damage": 20, "message": "..." }`

### 🏆 Блок 3: Лідерборд (Рейтинги)
- **`GET /api/leaderboard/users`**
  - *Опис:* Топ-10 найкращих студентів на платформі за кількістю XP.
- **`GET /api/leaderboard/universities`**
  - *Опис:* Рейтинг університетів (сумарний XP студентів кожного ВНЗ).

### 💬 Блок 4: Приватні Чати
- **`POST /api/chat`**
  - *Опис:* Відправити приватне текстове повідомлення іншому користувачу.
  - *Body:* `{ "senderId": "...", "receiverId": "...", "content": "Привіт!" }`
- **`GET /api/chat/history?user1Id=...&user2Id=...`**
  - *Опис:* Отримати історію переписки між двома користувачами.
- **`GET /api/chat/my?userId=...`**
  - *Опис:* Отримати список унікальних діалогів (користувачів, з якими була переписка) та їх останні повідомлення.

### 🛒 Блок 5: Магазин (Shop)
- **`GET /api/shop/items`**
  - *Опис:* Отримати список всіх доступних товарів (титули, підказки).
- **`POST /api/shop/buy`**
  - *Опис:* Купити товар за монети. Сервер перевіряє баланс і наявність титулу.
  - *Body:* `{ "userId": "...", "itemId": "..." }`

### 🎓 Блок 6: Інструментарій Викладача
Викладачі мають можливість відслідковувати успішність та створювати власні кастомні випробування замість нудних контрольних робіт.
- **`GET /api/teacher/stats?teacherId=...`**
  - *Опис:* Аналітика для викладача. Показує загальну кількість створених ним босів, відсоток успіху (Success Rate) та кількість унікальних студентів, які спробували вирішити завдання.
- **`POST /api/teacher/boss`**
  - *Опис:* Створення нового кастомного боса викладачем з власними пазлами-завданнями. Сервер автоматично генерує 6-значний `inviteCode`, який викладач дає студентам.
  - *Body:* `{ "creatorId": "...", "name": "Тест з ООП", "maxHp": 1000, "puzzles": [...] }`

---

## 📰 Блок 7: Форум (Reddit-style)
- **`GET /api/posts?universityId=...`**
  - *Опис:* Список постів (тредів) університету.
- **`POST /api/posts`**
  - *Опис:* Створити новий тред на форумі.
  - *Body:* `{ "title": "...", "content": "...", "userId": "...", "universityId": "..." }`
- **`GET /api/posts/:id`**
  - *Опис:* Отримати деталі поста та всі його коментарі.

## 🚀 Деплой (Render.com)
1. Підключіть GitHub репозиторій до Render.
2. Build Command: `bun install && bunx prisma generate && bun run db:push`
3. Start Command: `bun src/server.ts`
4. Додайте змінну: `DATABASE_URL` = ваш URL бази даних.
